# 商店折扣

## 题目描述

商店正在开展促销活动，给出了两种方案的折扣优惠。
第一种方案是购物满 x 元减 y 元；第二种方案是直接打 n 折（即价格变为原先的 n/10）。
这里的 x、y、n 均为正整数，并且 x > y，0 < n < 10。

需要注意的是，第一种方案中的满减优惠只能使用一次。例如购物满 x 元减 y 元时，若挑选了价格总和为 p 元的物品，只能减免 y 元，需要支付 p - y 元。

小明在商店挑选了价格总和为 p 元的物品，结账时只能使用一种优惠方案。小明最少需要支付多少钱？

## 输入格式

输入共四行，分别为四个正整数 x, y, n, p：

- 第1行：整数 x（1 ≤ x ≤ 10000）
- 第2行：整数 y（1 ≤ y ≤ 1000，且 x > y）
- 第3行：整数 n（1 ≤ n ≤ 9，表示打 n 折）
- 第4行：整数 p（1 ≤ p ≤ 10000，表示购物总额）

## 输出格式

输出一行，一个小数，表示小明最少需要支付多少钱，保留两位小数。

## 样例


```input1
8
7
9
10
```

```output1
3.00
```

## 样例解释
第一种方案：满 8 元减 7 元，p=10 满足条件，实际支付 10-7=3。
第二种方案：打 9 折，支付 10 * 9 / 10 = 9。最少为 3.00。

---

```input2
8
7
2
11
```

```output2
2.20
```

第一种方案：11 ≥ 8，支付 11-7 = 4。
第二种方案：打 2 折，支付 11 * 2 / 10 = 2.2。最少为 2.20。

## 数据范围

对于所有测试点，保证：
- x ≤ 10000
- y ≤ 1000
- 1 ≤ n ≤ 9
- p ≤ 10000

时限：1 秒，内存：256 MB。
